# Information Architecture & User Flow

## Site Map
1. **Login/Auth**: Secure enterprise SSO.
2. **Dashboard (Home)**: Focus on "What's Next" and "What's Pending".
3. **Meetings List**: Calendar and history.
4. **Meeting Detail (The Hub)**:
   - Agenda View
   - Document Viewer
   - Resolutions & Voting
   - Real-time Note taking
5. **Action Manager**: Tracking strategic outcomes.
6. **Governance Library**: Static document repository (Policies, Minutes).

## User Flow: "Preparing for a Meeting"
1. Arthur receives a notification: "Board Pack Ready".
2. Opens **Dashboard** on iPad; sees the next meeting at the top.
3. Clicks **Review Materials**; enters **Meeting Detail**.
4. Scans **Agenda**; clicks an item to load the associated PDF.
5. In the **Document Viewer**, adds a private note or highlights a section.
6. Sees a **Resolution** attached to the item; pre-votes "In Favor".

## User Flow: "During the Meeting"
1. Arthur opens the **Meeting Detail** page.
2. Follows the **Agenda** (Active item is synchronized across attendees).
3. Opens **Action Items** sidebar and assigns a task to Sarah directly from a discussion point.
4. Formal **Vote** is called; Arthur confirms his pre-vote via a one-click biometric prompt.
